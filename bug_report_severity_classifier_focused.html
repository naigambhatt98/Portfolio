Section 03

Severity Reference

A quick reference for the four severity levels.

Critical
Immediate attention
Crash, outage, major data loss, or serious security issue.
Handle first.
High
Urgent priority
Key functionality is broken or many users are affected.
Prioritize early.
Medium
Planned priority
Noticeable impact, but the system is still usable.
Plan in backlog.
Low
When capacity allows
Minor visual issues or low-impact edge cases.
Defer if needed.
