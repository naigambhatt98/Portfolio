03

The mathematics

Neural networks are, at their core, a composition of simple mathematical operations. These six equations describe everything that happens during training.

Forward pass

Weighted Sum

z = Σ wᵢxᵢ + b

Each neuron computes a weighted sum of its inputs plus a bias term. This is just a dot product — linear algebra.

Nonlinearity

Activation (ReLU)

a = max(0, z)

ReLU passes positive values unchanged and zeros out negatives. Simple, yet powerful enough for deep networks to learn complex functions.

Output layer

Sigmoid

σ(z) = 1 / (1 + e⁻ᶻ)

Squashes any real number to (0,1), making it useful for binary probability outputs. Softmax generalizes this to multiple classes.

Error measurement

Binary Cross-Entropy

L = −[y·log(ŷ) + (1−y)·log(1−ŷ)]

Penalizes confident wrong predictions harshly. When the model is certain and correct, loss approaches zero.

Learning rule

Gradient Descent

w ← w − η · ∂L/∂w

η is the learning rate — how big a step to take. Too large and training oscillates; too small and it crawls.

Credit assignment

Chain Rule

∂L/∂w = ∂L/∂a · ∂a/∂z · ∂z/∂w

Backpropagation applies this recursively through every layer to compute each weight's contribution to the total loss.

05

Key insights

Hands-on experimentation converts abstract theory into observable, intuitive behavior.

01

Depth unlocks complexity

Each hidden layer composes the previous one's features, enabling exponentially richer representations with only linear growth in parameters.

02

Nonlinearity is essential

Without activation functions, a 100-layer network is mathematically identical to a single linear layer. ReLU is what makes depth meaningful.

03

Weights encode knowledge

Everything a trained model has learned is stored in its weight matrix. Transfer learning works by reusing these weights on new tasks.

04

Noise degrades signal

Noisy labels give the optimizer contradictory gradients. The network wastes capacity memorizing noise instead of learning true structure.

05

Learning rate sets the pace

η controls the tradeoff between speed and precision. Modern optimizers like Adam adapt η per-parameter, sidestepping manual tuning.

06

Loss is the objective function

The choice of loss function encodes exactly what the model is optimizing. Mean squared error and cross-entropy lead to very different behaviors.

06

Summary

What we covered

  • Layers organize the flow of information from input to prediction
  • Neurons compute z = Σwᵢxᵢ + b — a parameterized linear function
  • Activation functions (ReLU, sigmoid) introduce essential nonlinearity
  • Weights are the learnable parameters — everything learned lives here
  • Loss functions (cross-entropy, MSE) quantify prediction error as a scalar
  • Gradient descent + backpropagation iteratively minimizes that loss
  • Architecture, noise, and learning rate jointly determine training success

Why this matters

Neural networks are function approximators — given enough data, depth, and compute, they can approximate any continuous function to arbitrary precision (Universal Approximation Theorem). What makes them powerful isn't magic, it's the combination of stacked linear transformations, nonlinear activations, and billions of gradient descent steps — repeated until the loss can't shrink further.

Understanding each component individually is what separates practitioners who can debug and improve models from those who treat them as black boxes.
